update_config(out_dir, overwrite='false', loopFilename=None, bbox_3d={'base': - 10000, 'maxx': 0, 'maxy': 0, 'minx': 0, 'top': 1200}, dtm_crs={'init': 'EPSG:4326'}, proj_crs=None, step_out=None, quiet='None', clut_path='', model_engine='loopstructural', run_flags=None, **kwargs)

Creates a ‘sub-project’ Config object and pre-processes input data for some area.

Parameters

  • out_dir (string) – Path to write output files to.

  • overwrite (string, optional) – Allow overwriting the given out_dir if it exists, false, true or in-place, defaults to false

  • bbox_3d (dict, optional) – 3D bounding box of coordinates and base/top values defining the area, defaults to { “minx”: 0, “maxx”: 0, “maxx”: 0, “maxy”: 0, “base”: -10000, “top”: 1200, }

  • dtm_crs (dict, optional) – Set the projection of the dtm, defaults to {‘init’: ‘EPSG:4326’}

  • proj_crs (dict, optional) – Set the projection of the input data, defaults to None

  • step_out (int, optional) – How far to consider outside the re-projected dtm, defaults to None

  • quiet (string, optional) – Allow or block print statements and matplotlib figures, ‘None’ to quiet nothing, ‘all’ to quiet everything, ‘no-figures’ to disable plots and allow text output. Defaults to ‘None’

  • clut_path (string, optional) – Path to custom map colours file

  • model_engine (string, optional) – Which modelling engine to use and set associated flags for, defaults to loopstructural

  • run_flags (dict, optional) – Global dictionary that defines custom params such as decimates and fault length see https://github.com/Loop3D/map2loop-2/issues/56

  • **kwargs –
